# Базовые принципы написания и разделение ответсвенности


## Менеджеры
1. Могут взаимодействовать напрямую с сущностями (помимо них с сущностями напрямую взаимодействуют только клетки и их CellEvents). Имеют не константные указатели на сущности 
2. Создают интерфейс взаимодействия контроллеров с сущностями
3. Только менеджеры помимо филда имеют не константую ссылку на экземпляр поля
4. В менеджерах проверяются только критичные условия, т.е. только те которые могут привести к крашу программы, т.е. например при атаке по противнику в ктетке мы проверяем, что противник в клетке есть, но не проверяем можем ли мы до неё дотянуться, тк это зона ответсвенности контроллера. Тоже самое относится к полю и клеткам


## Контроллеры
1. Не могут взаимодействовать напрямую с сущностями, работают с ними только через менеджеров
2. Имеют только константную ссылку на экземпляр поля
3. Определяют всю логику работы


## Поле
1. Основное игровое поле, содержит двумерный массив клеток
2. Все методы кроме метода получения не константной ссылки на вектор векторов клеток - константны и не изменяют состояние поля
3. Непосредственный досутп к полю (не const ссылку) имеют менеджеры и фабрики


## Клетка
1. Содержит в себе указатель на сущность если она на ней стоит, содержит уникальный указатель на событие клетки если таковое есть
2. Имеет 6 неконстантный метода изменяющих внутреннее состояние клетки или сущности находящиеся на ней. Вот они:
    - добавить сущность в клетку
    - удалить сущность из клетки
    - добавить CellEvent
    - нанести урон по клетки (если в ней заведомо есть сущность)
    - воздействовать на сущность CellEvent-ом клетки (если в ней заведомо есть сущность и CellEvent)
3. Имеет 4 константных метода для проверки состояния клетки
    - Есть ли сущность в клетке
    - Узнать тип клетки
    - Узнать тип сущности в клетки (сущность заведомом есть в клетке)
    - Есть ли CellEvent на клетке
